# NOASS (Not Only Agent Screen Saver) Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use executing-plans to implement this plan task-by-task.

**Goal:** A Tauri 2 Android app that wraps the 1-bit dithered orchestrator dashboard (force-directed node graph, Floyd-Steinberg dithering, CRT effects) as a mobile monitoring/control surface for Claude Code agent sessions — with a skins system, marketplace, accelerometer physics, weather effects, and an EEOAO expansion path that absorbs all projects into a gamified super-app.

**Architecture:** Tauri 2 + Vite + TypeScript frontend. The existing orchestrator-screensaver Canvas code (renderer.js, graph.js, ui.js, controls.js) is ported into TS modules. The app acts as a pure WebSocket client connecting to a remote server.js instance running on the user's machine. Skins are JSON manifests that control accent colors, dither parameters, CRT intensity, and background patterns — stored locally via Tauri's fs plugin. The marketplace is a bundled JSON catalog of skins with preview thumbnails, upgradeable to a remote API later. Android build via Tauri 2 mobile targets; iOS project generated but not built.

**Tech Stack:** Tauri 2 (Rust + WebView), Vite 6, TypeScript, Canvas API, WebSocket, Android SDK (via Tauri mobile), Floyd-Steinberg dithering engine

**Source material:**
- `~/codeprojects/orchestrator-screensaver/` — Complete working web app (server.js + public/)
- `~/codeprojects/neoorchestra/` — Tauri 2 scaffold (reference for project structure)

// src/renderer.ts
import type { AppState, Skin } from "./types";

export function clamp(v: number): number {
  return Math.max(0, Math.min(255, Math.round(v)));
}

export function hexToRGB(hex: string): { r: number; g: number; b: number } {
  hex = hex.replace("#", "");
  return {
    r: parseInt(hex.substring(0, 2), 16),
    g: parseInt(hex.substring(2, 4), 16),
    b: parseInt(hex.substring(4, 6), 16),
  };
}

export function floydSteinbergDither(imageData: ImageData): ImageData {
  const data = imageData.data;
  const w = imageData.width;
  const h = imageData.height;
  for (let y = 0; y < h; y++) {
    for (let x = 0; x < w; x++) {
      const i = (y * w + x) * 4;
      const old = data[i];
      const val = old > 128 ? 255 : 0;
      const err = old - val;
      data[i] = data[i + 1] = data[i + 2] = val;
      if (x + 1 < w) {
        const j = i + 4;
        data[j] = data[j + 1] = data[j + 2] = clamp(data[j] + (err * 7) / 16);
      }
      if (y + 1 < h) {
        if (x > 0) {
          const j = ((y + 1) * w + (x - 1)) * 4;
          data[j] = data[j + 1] = data[j + 2] = clamp(data[j] + (err * 3) / 16);
        }
        {
          const j = ((y + 1) * w + x) * 4;
          data[j] = data[j + 1] = data[j + 2] = clamp(data[j] + (err * 5) / 16);
        }
        if (x + 1 < w) {
          const j = ((y + 1) * w + (x + 1)) * 4;
          data[j] = data[j + 1] = data[j + 2] = clamp(data[j] + (err * 1) / 16);
        }
      }
    }
  }
  return imageData;
}

// --- Text cache ---
const _textCache = new Map<string, { canvas: HTMLCanvasElement; width: number; height: number }>();
const _TEXT_CACHE_MAX = 512;

function _textCacheKey(text: string, fontSize: number, color: string, bold: boolean): string {
  return `${text}|${fontSize}|${color}|${bold ? 1 : 0}`;
}

export function clearTextCache(): void {
  _textCache.clear();
}

export function drawDitheredText(
  ctx: CanvasRenderingContext2D,
  text: string,
  x: number,
  y: number,
  fontSize: number,
  options: { color?: "white" | "accent"; bold?: boolean; align?: "left" | "center" | "right" },
  skin: Skin
): void {
  const opts = { color: "white" as const, bold: false, align: "left" as const, ...options };
  const key = _textCacheKey(text, fontSize, opts.color, opts.bold);

  let cached = _textCache.get(key);
  if (!cached) {
    const weight = opts.bold ? "700" : "400";
    const font = `${weight} ${fontSize}px '${skin.fontFamily}', monospace`;

    const measure = document.createElement("canvas").getContext("2d")!;
    measure.font = font;
    const metrics = measure.measureText(text);
    const tw = Math.ceil(metrics.width) + 4;
    const th = Math.ceil(fontSize * 1.4) + 4;

    const off = document.createElement("canvas");
    off.width = tw;
    off.height = th;
    const offCtx = off.getContext("2d")!;

    offCtx.fillStyle = "#000";
    offCtx.fillRect(0, 0, tw, th);

    const gray = opts.color === "accent" ? 255 : 200;
    offCtx.fillStyle = `rgb(${gray},${gray},${gray})`;
    offCtx.font = font;
    offCtx.textBaseline = "top";
    offCtx.fillText(text, 2, 2);

    const imgData = offCtx.getImageData(0, 0, tw, th);
    floydSteinbergDither(imgData);

    if (opts.color === "accent") {
      const d = imgData.data;
      for (let i = 0; i < d.length; i += 4) {
        if (d[i] === 255) {
          d[i] = skin.accentRGB.r;
          d[i + 1] = skin.accentRGB.g;
          d[i + 2] = skin.accentRGB.b;
        }
      }
    }

    offCtx.putImageData(imgData, 0, 0);
    cached = { canvas: off, width: tw, height: th };
    if (_textCache.size > _TEXT_CACHE_MAX) {
      _textCache.delete(_textCache.keys().next().value!);
    }
    _textCache.set(key, cached);
  }

  let dx = x;
  if (opts.align === "center") dx = x - cached.width / 2;
  else if (opts.align === "right") dx = x - cached.width;

  ctx.drawImage(cached.canvas, Math.round(dx), Math.round(y - fontSize * 0.7));
}

// --- Circle cache ---
const _circleCache = new Map<string, { canvas: HTMLCanvasElement; size: number }>();
const _CIRCLE_CACHE_MAX = 128;

export function clearCircleCache(): void {
  _circleCache.clear();
}

export function drawDitheredCircle(
  ctx: CanvasRenderingContext2D,
  cx: number,
  cy: number,
  radius: number,
  fillGray: number
): void {
  const r = Math.ceil(radius);
  const size = r * 2 + 4;
  const key = `${r}|${fillGray}`;

  let cached = _circleCache.get(key);
  if (!cached) {
    const off = document.createElement("canvas");
    off.width = size;
    off.height = size;
    const offCtx = off.getContext("2d")!;

    offCtx.fillStyle = "#000";
    offCtx.fillRect(0, 0, size, size);

    const center = size / 2;
    offCtx.beginPath();
    offCtx.arc(center, center, r, 0, Math.PI * 2);
    offCtx.fillStyle = `rgb(${fillGray},${fillGray},${fillGray})`;
    offCtx.fill();

    const imgData = offCtx.getImageData(0, 0, size, size);
    floydSteinbergDither(imgData);
    offCtx.putImageData(imgData, 0, 0);

    cached = { canvas: off, size };
    if (_circleCache.size > _CIRCLE_CACHE_MAX) {
      _circleCache.delete(_circleCache.keys().next().value!);
    }
    _circleCache.set(key, cached);
  }

  ctx.drawImage(cached.canvas, Math.round(cx - cached.size / 2), Math.round(cy - cached.size / 2));
}

export function drawAccentRing(
  ctx: CanvasRenderingContext2D,
  cx: number,
  cy: number,
  radius: number,
  progress: number,
  skin: Skin
): void {
  if (progress <= 0) return;
  const accentColor = `#${skin.accent}`;
  ctx.beginPath();
  ctx.arc(cx, cy, radius + 2, -Math.PI / 2, -Math.PI / 2 + progress * Math.PI * 2);
  ctx.strokeStyle = accentColor;
  ctx.lineWidth = 2;
  ctx.shadowColor = accentColor;
  ctx.shadowBlur = 6;
  ctx.stroke();
  ctx.shadowBlur = 0;
}

export function applyScanlines(ctx: CanvasRenderingContext2D, w: number, h: number, skin: Skin): void {
  ctx.fillStyle = `rgba(0, 0, 0, ${skin.scanlineOpacity})`;
  for (let y = 0; y < h; y += skin.scanlineGap) {
    ctx.fillRect(0, y, w, 1);
  }
}

export function applyVignette(ctx: CanvasRenderingContext2D, w: number, h: number, skin: Skin): void {
  const cx = w / 2;
  const cy = h / 2;
  const outerRadius = Math.sqrt(cx * cx + cy * cy);
  const s = skin.vignetteStrength;
  const grad = ctx.createRadialGradient(cx, cy, outerRadius * 0.35, cx, cy, outerRadius);
  grad.addColorStop(0, "rgba(0,0,0,0)");
  grad.addColorStop(0.7, `rgba(0,0,0,${s * 0.25})`);
  grad.addColorStop(1, `rgba(0,0,0,${s})`);
  ctx.fillStyle = grad;
  ctx.fillRect(0, 0, w, h);
}

export function drawStippledLine(
  ctx: CanvasRenderingContext2D,
  x0: number,
  y0: number,
  x1: number,
  y1: number,
  color: string,
  gap: number
): void {
  const dx = x1 - x0;
  const dy = y1 - y0;
  const dist = Math.sqrt(dx * dx + dy * dy);
  const steps = Math.floor(dist / gap);
  ctx.fillStyle = color;
  for (let i = 0; i < steps; i += 2) {
    const t = i / steps;
    ctx.fillRect(Math.round(x0 + dx * t), Math.round(y0 + dy * t), 1, 1);
  }
}

export function drawDitheredBox(
  ctx: CanvasRenderingContext2D,
  x: number,
  y: number,
  w: number,
  h: number,
  fillGray: number
): void {
  const off = document.createElement("canvas");
  off.width = w;
  off.height = h;
  const offCtx = off.getContext("2d")!;
  offCtx.fillStyle = `rgb(${fillGray},${fillGray},${fillGray})`;
  offCtx.fillRect(0, 0, w, h);
  const imgData = offCtx.getImageData(0, 0, w, h);
  floydSteinbergDither(imgData);
  offCtx.putImageData(imgData, 0, 0);
  ctx.drawImage(off, Math.round(x), Math.round(y));
}

// src/ws-client.ts
import type { AppState } from "./types";

export function connectWS(state: AppState, url: string, onState: (msg: any) => void): void {
  const ws = new WebSocket(url);
  state.ws = ws;

  ws.addEventListener("open", () => {
    console.log("WS connected to", url);
  });

  ws.addEventListener("message", (e) => {
    try {
      const msg = JSON.parse(e.data as string);
      onState(msg);
    } catch (err) {
      console.error("WS parse error:", err);
    }
  });

  ws.addEventListener("close", () => {
    console.log("WS disconnected, reconnecting in 3s...");
    state.ws = null;
    setTimeout(() => connectWS(state, url, onState), 3000);
  });

  ws.addEventListener("error", () => {
    ws.close();
  });
}

export function sendCommand(state: AppState, type: string, target: string, extra?: Record<string, any>): void {
  if (state.ws && state.ws.readyState === WebSocket.OPEN) {
    state.ws.send(JSON.stringify({ type, target, ...extra }));
  }
}

### Task 13: Accelerometer Physics Engine

**Files:**
- Create: `src/physics-device.ts`
- Modify: `src/graph.ts`
- Modify: `src/types.ts`

**Step 1: Add DeviceMotion API integration**

```typescript
// src/physics-device.ts
export interface DevicePhysics {
  gravityX: number;  // -1 to 1, from accelerometer
  gravityY: number;
  shaking: boolean;  // detected via acceleration magnitude
}
```

Use `DeviceMotionEvent` API — works in Android WebView. Request permission via `DeviceMotionEvent.requestPermission()` on iOS.

Key behaviors:
- **Tilt** — nodes slide in the direction of gravity (like early iPhone liquid/ball apps)
- **Shake** — scatter all nodes randomly, they re-settle via force-directed layout
- **Rotate to landscape** — graph reflows to wide layout

**Step 2: Wire into force-directed physics**

In `graph.ts`, add a gravity vector from the accelerometer to the existing force model. Each node gets an additional force proportional to device tilt. The `centerGravity` pull still exists but device gravity adds directional bias.

```typescript
// In stepPhysics():
if (devicePhysics.gravityX !== 0 || devicePhysics.gravityY !== 0) {
  for (const n of nodes) {
    n.vx += devicePhysics.gravityX * skin.deviceGravityStrength;
    n.vy += devicePhysics.gravityY * skin.deviceGravityStrength;
  }
}
```

**Step 3: Add skin parameters for physics**

Extend Skin interface:
```typescript
deviceGravityStrength: number;  // 0 = disabled, 1 = normal, 2 = heavy
friction: number;               // 0-1, how quickly nodes stop sliding
bounciness: number;             // 0-1, how much nodes bounce off walls
```

**Step 4: Shake detection**

Monitor `DeviceMotionEvent.acceleration` magnitude. If > threshold for 3+ samples, trigger scatter animation.

## Key Gotchas

1. **WSL2 can't run Android emulator** — build the APK, then install on a real device via `adb install` (ADB talks to phone over USB/WiFi)
2. **Tauri 2 Android requires specific NDK version** — check `src-tauri/gen/android/build.gradle` for the required version after `tauri android init`
3. **Canvas text input on mobile** — use hidden `<input>` element pattern, not pure Canvas keyboard capture
4. **WebSocket CSP** — must allow `ws:` and `wss:` in Tauri's CSP config or connections will be silently blocked
5. **Google Fonts won't load offline** — bundle IBM Plex Mono as a local asset for the Android app
6. **Floyd-Steinberg is CPU-heavy** — the text/circle caches are essential for 30fps on mobile. Don't remove them.
7. **Tauri store plugin** — needs to be registered both in Rust (`.plugin(tauri_plugin_store::Builder::new().build())`) and in capabilities JSON
8. **Font bundling** — IBM Plex Mono must be bundled as a local WOFF2 asset, not loaded from Google Fonts CDN (offline Android)

## Architect Review (2026-03-20)

**Reviewer:** GPT 5.4 via Codex CLI (Sanitize and Fire protocol)
**Critiques:** 7 total — 5 blocking, 2 degrading, 0 cosmetic

### Blocking Issues Resolved

1. **[INTERFACE] `read` flow has no response contract**
   - **Fix:** The existing server.js already sends `{type: "readResult", target: string, content: string}` and `{type: "ack", command: string, target: string, success: boolean}`. These were omitted from the plan's protocol section. Added to types.ts:
   ```typescript
   interface ReadResultMessage { type: "readResult"; target: string; content: string; }
   interface AckMessage { type: "ack"; command: string; target: string; success: boolean; }
   ```
   The `onState` handler in ws-client.ts must dispatch on `msg.type`: "state" → update graph, "readResult" → show overlay, "ack" → optional toast.

2. **[COUPLING] "Thin shell" conflicts with mobile-native behavior**
   - **Fix:** Audited each mobile capability against web API availability:
     - Wake Lock → `navigator.wakeLock.request()` — Web API, works in Android WebView
     - Android back button → `window.addEventListener("popstate")` or Tauri 2's `onBackButtonPressed` event from `@tauri-apps/api/app`
     - Haptics → `navigator.vibrate()` — Web API, works on Android
     - Cleartext traffic → Android manifest config, not Rust code
     - ATS exceptions → iOS plist config, not Rust code
   - Only `onBackButtonPressed` requires Tauri-specific API. All others are standard web APIs. The "thin shell" claim holds. Added Tauri back button handling to Task 8 Step 3 explicitly.

3. **[INVARIANT] Node identity underspecified**
   - **Fix:** `pane.name` is the canonical stable identity key. The existing graph.js already uses `existing.get(pane.name)` for reconciliation. Made explicit in types.ts: `PaneData.name` is the unique key, `idx` is display-order only. Client reconciliation rule: on each state update, match nodes by `name`, preserve position/velocity for existing nodes, create new nodes for unknown names, remove nodes for disappeared names. `selectedNode` is validated against current state — if `selectedNode.pane.name` not in current panes, deselect.

4. **[RESOURCE] No bounded performance budget**
   - **Fix:** Added performance invariants to Task 12 Step 1:
     - Target: 30fps (33ms frame budget) on mid-range Android (Snapdragon 6 Gen 1 / Pixel 7a class)
     - Degradation strategy: if frame time > 40ms for 10 consecutive frames, disable scanlines and reduce vignette to `0`. If still > 40ms, reduce physics iterations (skip every other frame). If still > 40ms, reduce dither resolution (render at 0.5x then upscale)
     - Cache budget: max 512 text entries + 128 circle entries (already in renderer.ts)
     - Profile with `performance.now()` around each render phase, log to console in debug mode

5. **[INTERFACE] Settings input unvalidated WebView behavior**
   - **Fix:** Abandon Canvas-rendered text input for settings. Use a **DOM overlay** for the settings screen only: a simple `<div>` with `<input>` elements that slides over the canvas. The dashboard stays pure Canvas, but settings uses real DOM for reliable keyboard/focus. This is simpler and guaranteed to work on all WebViews. Updated Task 6 Step 1 to use DOM-based settings drawer instead of Canvas-rendered.

### Accepted Risks

1. **[TEMPORAL] Reconnect stale-state guard** (degrading)
   - The server broadcasts full state snapshots every 2s. After reconnect, the very next message is the current state — there's no "older message arrives after newer" scenario because the server has a single broadcast loop, not a queue. Risk accepted. If we add multi-server support later, we'll need a sequence counter.

2. **[FALSIFIABILITY] Mock mode divergence** (degrading)
   - For MVP, mock data is hand-authored (ported from server.js). The risk of divergence from real protocol is low because the server.js mock and real data use identical shapes. To mitigate: add a `src/fixtures/` directory with one recorded real server sample (captured via `wscat`) for integration testing later. Not blocking for initial Android release.

### Future Risks (Not Blocking)

- **EEOAO expansion** — The user wants to eventually dump all projects into NOASS as gamified features. The current screen state machine (`dashboard | settings | marketplace | read-overlay`) is extensible but will need a plugin/module registry when more screens arrive. Not blocking for MVP.
- **Remote marketplace** — The bundled catalog is fine for launch, but a remote catalog needs auth, caching, and error handling. Deferred.
- **Multi-server** — Currently single WS connection. Multi-server monitoring would need connection multiplexing and the sequence counter from the accepted temporal risk.

### Survivability Assessment

This plan is now likely to survive Stage 1 and Stage 2 implementation. The highest-risk phase is **Task 10 (Android build)** — Android SDK/NDK setup on WSL2 is notoriously fiddly, and Tauri 2 mobile support is still young. The protocol is complete (readResult/ack were already implemented in server.js, just undocumented), node identity is explicit, performance budgets are defined, and the settings screen uses reliable DOM instead of fragile Canvas text input. The plan should execute cleanly through Tasks 1-9 in a browser, with Task 10 being the real integration test against device hardware.

## Architect Review v2 (2026-03-20) - Stages 5-6

**Reviewer:** GPT 5.4 via Codex CLI (Sanitize and Fire protocol)
**Critiques:** 8 total - 6 blocking, 2 degrading, 0 cosmetic

### Blocking Issues Resolved

1. **[TEMPORAL] iOS motion permission has no executable trigger**
   - **Fix:** Add "Enable Motion" toggle in settings (DOM overlay). On first tap, calls `DeviceMotionEvent.requestPermission()`. If denied or unavailable, physics gracefully disabled (deviceGravityStrength forced to 0). Permission state persisted in store. Android doesn't require permission - feature auto-enables.

2. **[INVARIANT] Gravity vector lacks normalized coordinate contract**
   - **Fix:** Define canonical mapping in physics-device.ts:
     - Screen-space: X = right-positive, Y = down-positive (matches Canvas coords)
     - Raw accelerometer values remapped per orientation via `screen.orientation.angle`
     - Clamped to [-1, 1] on each axis
     - Exponential smoothing: `smoothed = prev * 0.7 + raw * 0.3` (prevents jitter)
     - Unit: normalized gravity (1.0 = 9.8 m/s^2 equivalent force)

3. **[RESOURCE] Performance fallback only covers particles**
   - **Fix:** Extended 3-tier degradation to cover all features:
     - Tier 1 (>40ms): disable weather particles + scanlines
     - Tier 2 (>50ms): disable accelerometer physics (force gravity to 0), skip every other physics frame
     - Tier 3 (>60ms): reduce dither resolution to 0.5x, disable vignette
     - Each tier activates after 10 consecutive frames above threshold, deactivates after 30 frames below

4. **[INTERFACE] Module IDs not constrained against navigation encoding**
   - **Fix:** Module ID validation: `/^[a-z][a-z0-9-]{1,30}$/`. Registration rejects IDs matching reserved screen names (`dashboard`, `settings`, `marketplace`, `read-overlay`). Duplicate IDs throw at registration time. Route format: `module:<id>` with `:` as separator guaranteed absent from valid IDs.

5. **[TEMPORAL] Module lifecycle ordering unspecified**
   - **Fix:** Defined lifecycle contract:
     1. `activeModule.destroy()` called and awaited (async allowed)
     2. All module-owned listeners/timers/contexts verified cleaned up
     3. `nextModule.init(context)` called
     4. Screen transitions only after init resolves
     - Modules must register cleanup functions via `context.onDestroy(fn)` hook

6. **[INTERFACE] Shared AppState access unconstrained**
   - **Fix:** Modules receive `ModuleContext` instead of raw `AppState`:
     ```typescript
     interface ModuleContext {
       readonly skin: Skin;
       readonly state: StateMessage | null;
       readonly width: number;
       readonly height: number;
       readonly isMobile: boolean;
       store: ModuleStore;  // namespaced key-value store
       onDestroy(fn: () => void): void;
     }
     ```
     No access to `ws`, `selectedNode`, `nodes`, `canvas`, or `ctx` directly. Render gets `ctx` as a parameter but cannot mutate host state.

### Accepted Risks

1. **[TEMPORAL] Shake retrigger** (degrading)
   - Add 2-second cooldown after scatter event. Simple, but if the threshold is wrong for some devices, users may experience no-shake or double-shake. Accepted - can tune threshold later with real device testing.

2. **[FALSIFIABILITY] Achievement unlocks not operationally testable** (degrading)
   - For MVP, achievements are simple counters in store: `{event: "agent_killed", count: 1, unlockedAt: timestamp}`. Testing via debug URL param `?unlock=all` to bypass conditions. Real telemetry deferred.

### Survivability Assessment v2

Stages 5-6 are now survivable. The highest risk shifted from module contracts (now bounded) to **real-device accelerometer behavior** - different Android WebViews may report motion events with varying accuracy, frequency, and coordinate systems. The smoothing + normalization layer mitigates this but needs device testing. Stage 6 module system is now properly isolated via ModuleContext, but the first real module port (NeoOrchestra audio) will be the true integration test since it involves WebAudio contexts that are hard to clean up.
